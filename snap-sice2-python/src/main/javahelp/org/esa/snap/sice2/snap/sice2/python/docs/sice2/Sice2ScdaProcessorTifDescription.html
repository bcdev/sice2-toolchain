SNAP Data Processors - SICE2 SCDA Processors Description

Processor Description

I/O Parameters Tab

Source Product Group

Name: The processor does not expect a dedicated source product. The sources are expected as single tif files in a specified directory (see processing parameters below).

Target Product Group

Name: Used to specify the name of the target product.

Save to: Used to specify whether the target product should be saved to the file system. The combo box presents a list of file formats.

Open in SNAP: Used to specify whether the target product should be opened in the Sentinel Toolbox. When the target product is not saved, it is opened in the Sentinel Toolbox automatically.


Processing Parameters Tab

Parameters

Input tif file directory:

The directoty containing the GeoTIFF input files. The following inputs are mandatory for the snow retrieval:

r_TOA_S1.tif, r_TOA_S5.tif, BT_S7.tif, BT_S8.tif, BT_S9.tif


The Processing Result

The retrieved cloud mask is written as band to the target product in the same way as in standard processing mode as listed in the table below:

Band name Meaning
scda_cloud_mask Cloud flag (see table below)


Compared to the standard processing mode, NDSI is not written here. The 'scda_cloud_mask' flag is not written as SNAP flag band, but keeps the original SICE2 toolchain values (see also [2]):

scda_cloud_mask value Description
0 Clear
1 Cloudy
255 Invalid


Note that the output is written into a single target product of selected format, as in the standard processing mode. This differs from the original SICE2 toolchain which always generates a single GeoTIFF file for the cloud mask. To exactly map this functionality, GeoTIFF should be selected as format for the target product.

SICE2 SCDA Cloud Detection Processors - Command Line Description

Command Line Description

Since the SICE2 SCDA Cloud Detection Processors make use of the SNAP graph processing framework, it can be used also as a command line tool. The graph processing is invoked by the command
  • ${SNAP-INSTALL-DIR}/bin/gpt
To obtain general help on the graph processing, use the command
  • ${SNAP-INSTALL-DIR}/bin/gpt -h
Specific help on the SICE2 SCDA Cloud Detection Processors can be obtained with
  • ${SNAP-INSTALL-DIR}/bin/gpt -h Sice2.Cloudmask.Slstr
  • ${SNAP-INSTALL-DIR}/bin/gpt -h Sice2.Cloudmask.Slstr.Tifs
Information on the usage and a list of all available parameters are given. The SICE2 SCDA Cloud Detection Processors parameters can also be specified via a graph xml file. E.g., a graph xml file for the SICE2 SCDA Cloud Detection Processor in standard mode could look like the following. 
    <graph id="sice2ScdaTest">
        <version>1.0</version>
        <node id="sice2ScdaNodeId">
            <operator>Sice2.Cloudmask.Slstr</operator>
            <sources>
                <source>${l1bProduct}</source>
            </sources>
            <parameters>
            </parameters>
        </node>
    </graph>
This graph can be invoked by the following call on the command line:
  • ${SNAP-INSTALL-DIR}/bin/gpt <graph-file.xml> -Sl1bProduct=<path-to-l1b-product>
